Duolca App Documentation

Duolca is NokiaEDU's portal to allow instructors to launch cloud VMs in Microsoft Azure. Find resources and documentation to help you run and build Duolca here.

What is Duolca?

Designed for instructors in NokiaEDU, Duolca is a portal to the Microsoft Azure Cloud Services Platform. It allows instructors to easily deploy their VMs.

How Do I Use It?

Deploy + Manage a Course

  1. Check the email sent you by NokiaEDU: contains COURSE NAME, DEPLOYMENT NAME, RESOURCE GROUP, and LOCATION NAME
  2. Input that information in the form on the home page, and click Deploy vLab
  3. Wait for about 10 minutes, before returning to the app and refreshing the index page:
    4. here
  4. Reenter the same information and click the Manage VLab button to view the Public IP Address!

Troubleshooting

Checking if your course has deployed

Typically, if you enter the correct values into the form and click Manage vLab, it will only redirect you if your deployment has completed. Otherwise, you’ll be redirected to the home screen. As of 1.0.0, you’ll have to access the Azure Management Portal (portal.azure.com) if you have permissions to view resources [you’ll go to Resource Groups > your resource group name > Deployment] to 100% confirm if your deployment was successful.

Error: ‘CREDENTIALS’ variable is not defined

>If you get this error, it’s likely because the application has not authenticated you as a user through the index page. Reload the index (duolcaapp.azurewebsites.net) and try again.

Error: After I click ‘Deploy vLab’, my page doesn’t redirect anywhere.

This is a known issue, and once ten minutes have passed, you should just refresh the index page and click on Manage vLab after re-entering your course info.

Error: Buttons (Start vLab, Stop vLab) on the Management Screen don’t work.

This may be an error on the coding side, please see NokiaEDU for support.
